## Bugs Fixed
- Body Arc Stretch: `ProgressView` struct name conflicted with SwiftUI's ProgressView → renamed to `ProgressTrackingView`
- Body Arc Stretch: AppIcon Contents.json had old multi-size format → updated to single universal format
- Pulse Rhythm Trainer & Step Forge Planner: tab bar icons not showing — SwiftUI `.tabItem` only supports Image/Text, custom Shape views are ignored → replaced with custom tab bar (VStack + HStack)
- Step Forge Planner: Builder stepper counters not changing — Stepper too narrow → replaced with custom +/- buttons
- Step Forge Planner: white screen on START — `.fullScreenCover(isPresented:)` race condition → changed to `.fullScreenCover(item:)` pattern
- Pulse Rhythm Trainer: no safe area under tab bar → removed `edgesIgnoringSafeArea(.bottom)`
- Step Forge Planner: History/Builder text overflow — fixed bar chart widths (adaptive), reduced spacing

## Lessons
- Master says no need to test apps, just verify build succeeds
- ProgressView is a reserved SwiftUI name — never use it for custom views
- Always use iPhone 17 for simulator destination (no iPhone 16 available)
- SwiftUI `.tabItem` ONLY supports `Image` and `Text` — custom views silently ignored. Always use custom tab bar with Shape icons
- Design spam risk: same color palette + same category + same structure = 4.3(a) risk. Master hasn't decided yet whether to differentiate or publish from different accounts
